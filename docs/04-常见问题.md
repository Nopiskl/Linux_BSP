## 🔧 环境相关

###  Q1: 交叉编译器未找到

**错误信息：**
```
scripts/Kconfig.include:39: compiler 'aarch64-linux-gnu-gcc' not found
```

**原因：** 未安装 ARM64 交叉编译工具链

**解决方法：**
```bash
# 安装交叉编译器
sudo apt-get update
sudo apt-get install gcc-aarch64-linux-gnu

# 验证安装
aarch64-linux-gnu-gcc --version
```

---

### Q2: mmdebstrap 命令不存在

**错误信息：**
```bash
./build-rootfs.sh: line XX: mmdebstrap: command not found
```

**解决方法：**
```bash
sudo apt-get install mmdebstrap debootstrap qemu-user-static
```

---

### Q3: dialog 界面显示异常

**现象：** 交互菜单显示乱码或无法操作

**解决方法：**
```bash
# 安装 dialog
sudo apt-get install dialog

# 确认终端支持
echo $TERM
# 应显示 xterm、xterm-256color 等

# 如果使用 SSH，添加：
export TERM=xterm-256color
```

---

### Q4: 磁盘空间不足

**错误信息：**
```
No space left on device
```

**检查方法：**
```bash
# 检查磁盘使用情况
df -h

# 检查具体目录占用
du -sh /path/to/BSP_T527/*
```

**解决方法：**
```bash
# 方法1：清理旧构建产物
rm -rf output/

# 方法2：清理 APT 缓存
sudo apt-get clean

# 方法3：清理 Docker（如果使用）
docker system prune -a

# 方法4：清理系统日志
sudo journalctl --vacuum-time=3d

# 方法5：删除不需要的内核
sudo apt autoremove --purge
```

## 🔨 编译相关

### Q5: 内核编译失败

**错误信息：**
```
make: *** [Makefile:XXX] Error 2
```

**排查步骤：**

#### 1. 检查依赖包
```bash
sudo apt-get install -y \
    make bc bison flex \
    libssl-dev libncurses-dev libelf-dev
```

#### 2. 清理后重新编译
```bash
cd output/linux
make ARCH=arm64 mrproper
cd ../..
sudo ./build.sh --board example --build-rootfs no
```

#### 3. 检查内核配置
```bash
cd output/linux
make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- menuconfig
# 检查是否有明显错误配置
```

#### 4. 查看详细错误
```bash
# 保存完整日志
sudo ./build.sh 2>&1 | tee build.log
# 搜索错误信息
grep -i "error" build.log
```

---

### Q6: 内核模块编译失败

**错误信息：**
```
ERROR: could not insert module XXX.ko
```

**解决方法：**
```bash
# 1. 确保内核配置正确
cd output/linux
make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- menuconfig
# 启用对应的模块支持

# 2. 重新编译
cd ../..
sudo ./build.sh --board example --build-rootfs no

# 3. 检查模块依赖
modinfo output/linux/path/to/module.ko
```

---

### Q7: 头文件安装时间过长

**现象：** `install_headers` 阶段耗时超过 10 分钟

**说明：** 这是正常的！头文件安装需要复制大量小文件（~36000+ 个文件）。

**优化方法：**
```bash
# 已经在代码中优化，耗时约 2-3 分钟
# 如果还是很慢，检查：

# 1. 是否使用 SSD
df -T /path/to/BSP_T527

# 2. 磁盘 I/O 性能
iostat -x 1
```

## 📦 RootFS 相关

### Q8: mmdebstrap 失败

**错误信息：**
```
E: Malformed entry in list file
```

**原因：** APT 源配置错误

**解决方法：**
```bash
# 检查镜像源格式
cat rootfs/ubuntu/jammy/apt-sources/sources.list

# 正确格式：
deb http://mirrors.aliyun.com/ubuntu-ports jammy main restricted universe multiverse

# 错误格式（有多余的路径）：
deb http://mirrors.aliyun.com/ubuntu-ports ubuntu/jammy main
```

---

### Q9: 软件包下载失败

**错误信息：**
```
E: Failed to fetch http://...
E: Unable to fetch some archives
```

**解决方法：**

#### 方法1：切换镜像源
```bash
# 使用其他镜像源重试
sudo ./build.sh -b example -v ubuntu/jammy -t cli \
  -m https://mirrors.tuna.tsinghua.edu.cn/ubuntu-ports \
  -r yes
```

#### 方法2：检查网络
```bash
# 测试连接性
ping -c 4 mirrors.aliyun.com
curl -I https://mirrors.aliyun.com/ubuntu-ports/

# 检查 DNS
nslookup mirrors.aliyun.com
```

#### 方法3：清理并重试
```bash
# 删除缓存的 RootFS
rm output/rootfs-ubuntu-jammy-cli.tar.gz

# 重新构建
sudo ./build.sh -b example -v ubuntu/jammy -t cli -r yes
```

### Q10: 软件包依赖冲突

**错误信息：**
```
The following packages have unmet dependencies:
 package-a: Depends: package-b (>= x.x.x) but x.x.x is to be installed
```

**解决方法：**
```bash
# 1. 检查包列表中是否有冲突的包
vim rootfs/ubuntu/jammy/packages/cli.list

# 2. 移除有问题的包，重新构建
# 或者等待上游修复

# 3. 使用更稳定的发行版版本
sudo ./build.sh -v ubuntu/jammy -t cli -r yes  # Jammy 更稳定
```

---

### Q11: 首次启动分区未扩展

**现象：** 系统启动后磁盘空间仍然很小

**检查方法：**
```bash
# 在目标板上执行
df -h
lsblk
```

**解决方法：**
```bash
# 手动扩展
sudo growpart /dev/mmcblk0 2
sudo resize2fs /dev/mmcblk0p2

# 或检查 init-resize 服务
systemctl status init-resize
journalctl -u init-resize
```

## 🌐 网络相关

### Q12: DNS 解析失败

**错误信息：**
```
Temporary failure in name resolution
Could not resolve 'archive.ubuntu.com'
```

**解决方法：**

#### 方法1：配置 DNS
```bash
sudo mkdir -p /etc/systemd/resolved.conf.d
sudo tee /etc/systemd/resolved.conf.d/dns.conf > /dev/null << 'EOF'
[Resolve]
DNS=8.8.8.8 114.114.114.114
FallbackDNS=223.5.5.5 223.6.6.6
EOF

sudo systemctl restart systemd-resolved

# 验证
nslookup google.com
```

#### 方法2：临时修改 resolv.conf
```bash
sudo tee /etc/resolv.conf > /dev/null << 'EOF'
nameserver 8.8.8.8
nameserver 114.114.114.114
nameserver 223.5.5.5
EOF
```

#### 方法3：使用国内 DNS
```bash
# 阿里 DNS
nameserver 223.5.5.5
nameserver 223.6.6.6

# 腾讯 DNS
nameserver 119.29.29.29

# DNSPod
nameserver 119.28.28.28
```

### Q13: 代理配置

**场景：** 需要通过代理访问网络

**配置方法：**
```bash
# 临时设置
export http_proxy="http://proxy.example.com:8080"
export https_proxy="http://proxy.example.com:8080"
export no_proxy="localhost,127.0.0.1"

# 执行构建
sudo -E ./build.sh

# 永久设置（添加到 ~/.bashrc）
cat >> ~/.bashrc << 'EOF'
export http_proxy="http://proxy.example.com:8080"
export https_proxy="http://proxy.example.com:8080"
EOF
```

## 🔐 权限相关

### Q15: 权限被拒绝

**错误信息：**
```
Permission denied
```

**解决方法：**
```bash
# 使用 sudo 运行构建脚本
sudo ./build.sh

# 或者修改 output 目录所有权（不推荐）
sudo chown -R $USER:$USER output/
```

---

### Q16: 无法写入 .config 文件

**错误信息：**
```
NoPermissions: Error: EACCES: permission denied, open '.config'
```

**原因：** 文件由 root 创建，当前用户无法修改

**解决方法：**
```bash
# 修改 output 目录所有权
sudo chown -R $USER:$USER output/

# 或使用 sudo 编辑
sudo vim output/linux/.config
```

---

### Q17: dpkg 锁被占用

**错误信息：**
```
E: Could not get lock /var/lib/dpkg/lock-frontend
```

**原因：** 其他 apt 进程正在运行

**解决方法：**
```bash
# 1. 等待其他进程完成
ps aux | grep -i apt

# 2. 如果卡住，强制解锁（谨慎）
sudo killall apt apt-get
sudo rm /var/lib/apt/lists/lock
sudo rm /var/cache/apt/archives/lock
sudo rm /var/lib/dpkg/lock*

# 3. 修复
sudo dpkg --configure -a
sudo apt-get update
```

## ⚡ 性能相关

### Q18: 构建速度太慢

**优化方法：**

#### 1. 使用国内镜像源
```bash
sudo ./build.sh -m https://mirrors.aliyun.com/ubuntu-ports -r yes
```

#### 2. 增加并行编译数
```bash
export MAKEFLAGS="-j$(nproc)"
sudo -E ./build.sh
```

#### 3. 使用 ccache
```bash
sudo apt-get install ccache
export PATH="/usr/lib/ccache:$PATH"
export CCACHE_DIR="$HOME/.ccache"
```

#### 4. 使用 SSD
将项目移到 SSD 分区可显著提升速度。

#### 5. 增加交换空间
```bash
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

---

### Q19: 内存不足

**错误信息：**
```
virtual memory exhausted: Cannot allocate memory
```

**解决方法：**
```bash
# 1. 减少并行编译数
export MAKEFLAGS="-j2"  # 而不是 -j$(nproc)

# 2. 增加交换空间（见 Q18）

# 3. 关闭不必要的程序
# 释放内存

# 4. 使用更小的系统类型
sudo ./build.sh -t cli -r yes  # CLI 而不是 GNOME
```

## 🐛 其他问题

### Q20: 脚本语法错误

**错误信息：**
```
./build.sh: line XXX: syntax error near unexpected token
```

**原因：** 可能是 Windows 行结尾（CRLF）

**解决方法：**
```bash
# 转换行结尾
sed -i 's/\r$//' build.sh
sed -i 's/\r$//' tools/*.sh

# 或使用 dos2unix
sudo apt-get install dos2unix
dos2unix build.sh tools/*.sh
```

---

### Q21: Git 克隆失败

**错误信息：**
```
fatal: unable to access 'https://github.com/...': Could not resolve host
```

**解决方法：**

#### 方法1：使用 Gitee 镜像
脚本会自动尝试 Gitee 镜像。

#### 方法2：手动克隆
```bash
# 手动下载源码
mkdir -p output
cd output
git clone --depth=1 https://github.com/torvalds/linux.git
cd ..

# 跳过源码下载步骤
sudo ./build.sh --board example --build-rootfs no
```

#### 方法3：使用代理
```bash
export https_proxy="http://proxy.example.com:8080"
sudo -E ./build.sh
```

---

### Q22: 构建中断后如何恢复

**场景：** 构建过程中断（网络、断电等）

**恢复方法：**
```bash
# 1. 检查 output 目录状态
ls -lh output/

# 2. 如果内核已构建完成
#    可以跳过内核构建
sudo ./build.sh --board example --kernel-only no --build-rootfs yes

# 3. 如果需要完全重新开始
rm -rf output/
sudo ./build.sh
```

### Q23: 查看构建日志

**方法：**
```bash
# 保存完整日志
sudo ./build.sh 2>&1 | tee build-$(date +%Y%m%d-%H%M%S).log

# 搜索错误
grep -i "error" build-*.log
grep -i "fail" build-*.log

# 查看最后 50 行
tail -50 build-*.log
```

**检查内核包：**

## 💡 预防问题的最佳实践

1. **充分准备** - 完整阅读环境配置文档
2. **稳定网络** - 使用有线网络或稳定的 Wi-Fi
3. **足够空间** - 确保至少 30GB 可用空间
4. **国内镜像** - 使用国内镜像源加速
5. **保存日志** - 使用 `tee` 保存构建日志
6. **逐步构建** - 先构建 CLI，再尝试桌面
7. **版本稳定** - 使用推荐的发行版版本
8. **定期更新** - 更新系统和依赖包
