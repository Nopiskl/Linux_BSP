## 📁 configs/ - 板级配置

### 结构

```
configs/
├── example.conf           # 示例板型配置
├── t527_dev.conf          # T527 开发板配置
└── README.md              # 配置说明
```

### 配置文件格式

**示例：configs/example.conf**
```bash
# 板型信息
BOARD_NAME="example"
ARCH="arm64"

# 内核配置
KERNEL_DEFCONFIG="defconfig"
KERNEL_REPO="https://github.com/torvalds/linux.git"
KERNEL_BRANCH="master"

# 引导加载器
BOOTLOADER_TYPE="uboot"
UBOOT_DEFCONFIG="xxx_defconfig"

# 设备树
DTB_FILE="xxx.dtb"
```

### 添加新板型

```bash
# 1. 复制示例配置
cp configs/example.conf configs/my_board.conf

# 2. 编辑配置
vim configs/my_board.conf

# 3. 使用新配置
sudo ./build.sh --board my_board
```

## 🔧 tools/ - 构建工具集

### 结构

```
tools/
├── get-sources.sh         # 源码获取
├── build-kernel.sh        # 内核构建
├── build-rootfs.sh        # RootFS 构建
├── build-boot.sh          # 引导加载器构建
└── README.md              # 工具说明
```

### 工具详解

#### get-sources.sh

**功能：** 智能获取和管理源码

**特性：**
- Git 自动克隆和更新
- 分支/标签切换
- Gitee 镜像回退
- 源码完整性验证

**调用方式：**
```bash
bash tools/get-sources.sh example
```

**关键函数：**
- `clone_linux()` - 克隆 Linux 内核
- `clone_uboot()` - 克隆 U-Boot
- `clone_atf()` - 克隆 ARM Trusted Firmware

---

#### build-kernel.sh

**功能：** 编译内核并生成 .deb 包

**流程：**
1. 加载板级配置
2. 编译内核和模块
3. 安装内核头文件
4. 打包成 Debian 软件包

**调用方式：**
```bash
bash tools/build-kernel.sh example
```

**输出：**
- `output/linux/` - 内核源码和构建产物
- `output/example-kernel-pkgs/*.deb` - Debian 软件包

**关键函数：**
- `build_kernel()` - 编译内核
- `build_modules()` - 编译模块
- `install_headers()` - 安装头文件（优化版，~2-3分钟）
- `create_deb_packages()` - 创建 .deb 包

---

#### build-rootfs.sh

**功能：** 构建根文件系统

**流程：**
1. 解析发行版和类型
2. 使用 mmdebstrap 创建基础系统
3. 安装软件包
4. 应用配置叠加层
5. 配置系统服务
6. 打包成 .tar.gz

**调用方式：**
```bash
bash tools/build-rootfs.sh -b example -v ubuntu/jammy -t cli
```

**参数：**
```
-b, --board BOARD          板型名称
-v, --version VERSION      发行版版本 (ubuntu/jammy)
-t, --type TYPE            系统类型 (cli, xfce, gnome, kde, lxqt)
-m, --mirror MIRROR        APT 镜像源 URL
```

**输出：**
- `output/rootfs-{distro}-{version}-{type}.tar.gz`

**关键函数：**
- `run_debootstrap()` - 创建基础系统
- `prepare_apt_sources()` - 配置 APT 源
- `setup_mount_resolv()` - 挂载必要文件系统
- `setup_firstrun()` - 配置首次启动服务
- `clean_rootfs()` - 清理临时文件

**相关文档：** [RootFS 配置](./03-RootFS配置.md)

---

#### build-boot.sh

**功能：** 构建引导加载器（U-Boot/ATF）

**流程：**
1. 加载板级配置
2. 编译 ARM Trusted Firmware（如需）
3. 编译 U-Boot
4. 生成引导镜像

**调用方式：**
```bash
bash tools/build-boot.sh example
```

**输出：**
- `output/bootloader-example/`

---

#### overlays/

**作用：** 系统配置文件叠加层

**工作原理：**
1. 构建时，overlays 中的文件会复制到 RootFS
2. 保持目录结构：`overlays/common/etc/hostname` → `/etc/hostname`
3. 支持任意系统配置文件

**示例：**
```bash
# 创建自定义主机名
echo "my-board" > rootfs/overlays/common/etc/hostname

# 添加自定义脚本
cat > rootfs/overlays/common/usr/local/bin/my-script.sh << 'EOF'
#!/bin/bash
echo "Custom script"
EOF
chmod +x rootfs/overlays/common/usr/local/bin/my-script.sh
```

**相关文档：** [RootFS 配置 - 系统配置](./03-RootFS配置.md#系统配置)

## 📤 output/ - 构建输出

**说明：** 此目录由构建脚本自动生成

### 完整结构

```
output/
├── linux/                         # 内核源码和构建产物
│   ├── .config                   # 内核配置
│   ├── arch/arm64/boot/
│   │   ├── Image                # 内核镜像
│   │   └── dts/                 # 设备树
│   ├── vmlinux                   # 未压缩内核（调试用）
│   └── ...
│
├── example-kernel-pkgs/           # 内核 Debian 包
│   ├── linux-image-*.deb        # 内核镜像包（60-80MB）
│   ├── linux-headers-*.deb      # 内核头文件（15-20MB）
│   └── linux-libc-dev-*.deb     # 内核开发库（1-2MB）
│
├── rootfs-ubuntu-jammy-cli.tar.gz        # RootFS 压缩包
├── rootfs-ubuntu-jammy-xfce.tar.gz
├── rootfs-debian-bookworm-cli.tar.gz
│
├── bootloader-example/            # 引导加载器（如有）
│   ├── u-boot.bin
│   └── ...
│
└── deb-data/                      # .deb 打包临时目录
    ├── image/                    # 内核镜像数据
    └── headers/                  # 头文件数据
```

### 文件用途

#### linux-image-*.deb

**内容：**
- 内核镜像 (`/boot/vmlinuz-*`)
- 内核模块 (`/lib/modules/*`)
- 设备树 (`/boot/dtbs/*`)

**用途：**
```bash
# 在目标板上安装
sudo dpkg -i linux-image-*.deb

# 自动配置引导加载器
# 自动生成 initramfs
```

---

#### linux-headers-*.deb

**内容：**
- 内核头文件 (`/usr/src/linux-headers-*`)
- 编译脚本和 Makefile
- 内核符号表

**用途：**
```bash
# 在目标板上安装
sudo dpkg -i linux-headers-*.deb

# 用于编译内核模块
cd /path/to/module
make
```

---

#### rootfs-*.tar.gz

**内容：** 完整的根文件系统

**用途：**
```bash
# 解压到 SD 卡
sudo tar -xzf rootfs-ubuntu-jammy-cli.tar.gz -C /mnt/sdcard/

# 或制作镜像文件
# ...
```

**命名规则：**
```
rootfs-{distro-type}-{distro-version}-{type}.tar.gz

示例：
rootfs-ubuntu-jammy-cli.tar.gz
rootfs-debian-bookworm-xfce.tar.gz
```

## 🔄 缓存机制

### RootFS 缓存

**机制：** 已构建的 RootFS 会自动缓存

**检测逻辑：**
```bash
# build.sh 会检查文件是否存在
ROOTFS_TARBALL="output/rootfs-ubuntu-jammy-cli.tar.gz"

if [ -f "${ROOTFS_TARBALL}" ]; then
    echo "✓ Found existing rootfs, skip build"
else
    # 执行构建
    bash tools/build-rootfs.sh
fi
```

**优势：**
- 二次构建时间从 30 分钟 → 5 分钟
- 支持多版本并存
- 手动删除可强制重建

**示例：**
```bash
# 首次构建（完整流程）
sudo ./build.sh -v ubuntu/jammy -t cli -r yes
# 耗时：~30 分钟

# 二次构建（使用缓存）
sudo ./build.sh -v ubuntu/jammy -t cli -r yes
# 输出：✓ Found existing rootfs tarball
# 耗时：~5 分钟

# 强制重建
rm output/rootfs-ubuntu-jammy-cli.tar.gz
sudo ./build.sh -v ubuntu/jammy -t cli -r yes
# 耗时：~30 分钟
```

## 📝 自定义和扩展

### 添加新工具脚本

```bash
# 1. 创建脚本
cat > tools/custom-tool.sh << 'EOF'
#!/bin/bash
```

### 添加新板型配置

```bash
# 1. 创建配置文件

# 2. 编辑配置
vim configs/my_board.conf

# 3. 使用
sudo ./build.sh --board my_board
```

### 添加新发行版版本

```bash
# 1. 创建目录结构
mkdir -p rootfs/ubuntu/oracular/{packages,apt-sources}

# 2. 复制配置模板
cp -r rootfs/ubuntu/jammy/packages/* rootfs/ubuntu/oracular/packages/
cp rootfs/ubuntu/jammy/apt-sources/sources.list rootfs/ubuntu/oracular/apt-sources/

# 3. 修改配置
sed -i 's/jammy/oracular/g' rootfs/ubuntu/oracular/apt-sources/sources.list

# 4. 使用
sudo ./build.sh -v ubuntu/oracular -t cli -r yes
```
