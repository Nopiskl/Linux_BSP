## 🔄 构建流程概览

```
┌─────────────────────────────────────────────┐
│              1. 获取源码                    │
│  • 下载 Linux 内核源码                      │
│  • 下载引导加载器                           │
│  • 使用 Git 自动管理                        │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│              2. 构建内核                    │
│  • 加载板级配置                             │
│  • 编译内核和模块                           │
│  • 生成 .deb 软件包                         │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│           3. 构建 RootFS（可选）            │
│  • 使用 mmdebstrap 创建基础系统             │
│  • 安装指定软件包                           │
│  • 配置系统服务                             │
│  • 打包成 .tar.gz                           │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│              4. 生成输出                    │
│  • output/example-kernel-pkgs/              │
│  • output/rootfs-*.tar.gz                   │
└─────────────────────────────────────────────┘
```

**⏱️ 总耗时：** 30-90 分钟（取决于配置和网络）

## 🎨 交互式构建（推荐）

### 1. 启动构建脚本

```bash
cd /path/to/BSP_T527
sudo ./build.sh
```

### 2. 配置向导

#### 步骤 1：选择板型

```
┌─── Select Board ───┐

│                     │
│  [  OK  ]  [Cancel]│
└─────────────────────┘
```

**说明：** 不同板型对应不同的硬件配置

#### 步骤 2：选择内核版本

```
┌─── Select Kernel ───┐
│                      │
│  (*) linux-6.1       │

│  ( ) linux-6.6       │

└──────────────────────┘
```

**推荐：** linux-6.1（稳定版）

## ⌨️ 命令行构建

### 基本语法

```bash
sudo ./build.sh [选项]
```

### 常用组合示例

#### 示例 1：最小 CLI 系统（推荐首次构建）

```bash
sudo ./build.sh \
  --board example \
  --kernel linux-6.1 \
  --distro-version ubuntu/jammy \
  --rootfs-type cli \
  --apt-mirror https://mirrors.aliyun.com/ubuntu-ports \
  --build-rootfs yes
```

**⏱️ 时间：** ~30 分钟

#### 示例 2：仅构建内核

```bash
sudo ./build.sh \
  --board example \
  --kernel linux-6.1 \
  --build-rootfs no
```

**⏱️ 时间：** ~10-15 分钟

#### 示例 3：XFCE 桌面系统

```bash
sudo ./build.sh \
  --board example \
  --distro-version ubuntu/jammy \
  --rootfs-type xfce \
  --apt-mirror https://mirrors.tuna.tsinghua.edu.cn/ubuntu-ports \
  --build-rootfs yes
```

**⏱️ 时间：** ~45-60 分钟

#### 示例 4：Debian 系统

```bash
sudo ./build.sh \
  --board example \
  --distro-version debian/bookworm \
  --rootfs-type cli \
  --apt-mirror https://mirrors.ustc.edu.cn/debian \
  --build-rootfs yes
```

#### 示例 5：使用缩写参数

```bash
sudo ./build.sh -b example -k linux-6.1 -v ubuntu/jammy -t cli -r yes
```

## 📝 构建选项说明

### 完整参数列表

```bash
./build.sh [OPTIONS]

必选参数：
  -b, --board BOARD                板型名称 (example, t527_dev)
  
内核相关：
  -k, --kernel KERNEL              内核版本 (linux-6.1, linux-5.15)
  -l, --kernel-only yes|no         是否仅构建内核 (默认: no)
  
RootFS 相关：
  -r, --build-rootfs yes|no        是否构建 RootFS (默认: no)
  -v, --distro-version VERSION     发行版版本 (ubuntu/jammy, debian/bookworm)
  -t, --rootfs-type TYPE           系统类型 (cli, xfce, gnome, kde, lxqt)
  -m, --apt-mirror MIRROR          APT 镜像源 URL
  
其他：
  -h, --help                       显示帮助信息
```

### 参数详解

#### --board（板型）

可用值：
- `example` - 示例板型（默认）
- `t527_dev` - T527 开发板
- 其他自定义板型（需在 `configs/` 目录配置）

#### --distro-version（发行版）

**Ubuntu 系列：**
- `ubuntu/focal` - Ubuntu 20.04 LTS
- `ubuntu/jammy` - Ubuntu 22.04 LTS ⭐ 推荐
- `ubuntu/noble` - Ubuntu 24.04 LTS

**Debian 系列：**
- `debian/bullseye` - Debian 11
- `debian/bookworm` - Debian 12 ⭐ 推荐
- `debian/trixie` - Debian Testing

#### --rootfs-type（系统类型）

| 类型 | 说明 | 包含组件 |
|------|------|---------|
| `cli` | 命令行 | 基础系统 + 常用工具 |
| `xfce` | XFCE 桌面 | CLI + XFCE + 浏览器 |
| `gnome` | GNOME 桌面 | CLI + GNOME + 完整办公套件 |
| `kde` | KDE Plasma | CLI + KDE + 多媒体工具 |
| `lxqt` | LXQt 桌面 | CLI + LXQt + 基础应用 |

#### --apt-mirror（镜像源）

**预设镜像：**
```bash
# 官方源
--apt-mirror official

# 阿里云（推荐）
--apt-mirror aliyun

# 清华大学
--apt-mirror tsinghua

# 中科大
--apt-mirror ustc

# 华为云
--apt-mirror huawei
```

**自定义源：**
```bash
# Ubuntu
--apt-mirror https://mirrors.example.com/ubuntu-ports

# Debian
--apt-mirror https://mirrors.example.com/debian
```

## 📦 构建产物说明

### 目录结构

```
output/
├── example-kernel-pkgs/              # 内核软件包
│   ├── linux-image-*.deb            # 内核镜像
│   ├── linux-headers-*.deb          # 内核头文件
│   └── linux-libc-dev-*.deb         # 内核开发库
│
├── rootfs-ubuntu-jammy-cli.tar.gz   # RootFS 压缩包
│
├── linux/                            # 内核源码和构建产物
│   ├── .config                      # 内核配置
│   ├── arch/arm64/boot/Image        # 内核镜像
│   └── ...
│
└── bootloader-example/               # 引导加载器（如有）
```

### 文件说明

#### 1. 内核软件包 (.deb)

```bash
ls -lh output/example-kernel-pkgs/

# 输出示例：
linux-image-6.1.x-arm64_xxx_arm64.deb       # 60-80MB  - 内核镜像和模块
linux-headers-6.1.x-arm64_xxx_arm64.deb     # 15-20MB  - 内核头文件
linux-libc-dev-6.1.x_xxx_arm64.deb          # 1-2MB    - 内核开发库
```

**用途：**
- 可直接在目标板上安装：`sudo dpkg -i *.deb`
- 用于构建内核模块

#### 2. RootFS 压缩包

```bash
ls -lh output/rootfs-*.tar.gz

# 输出示例：
rootfs-ubuntu-jammy-cli.tar.gz              # 250-300MB   - CLI 系统
rootfs-ubuntu-jammy-xfce.tar.gz             # 800MB-1.2GB - XFCE 桌面
rootfs-debian-bookworm-cli.tar.gz           # 200-250MB   - Debian CLI
```

**用途：**
- 制作 SD 卡镜像
- 部署到目标设备
- 作为基础系统二次开发

#### 3. 内核源码目录

```bash
output/linux/
├── .config                    # 内核配置文件
├── arch/arm64/boot/Image      # 原始内核镜像
└── vmlinux                    # 未压缩的内核（用于调试）
```

## 🚀 高级用法

### 自定义内核配置

```bash
# 1. 启动配置菜单
cd output/linux
make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- menuconfig

# 2. 修改配置后，重新构建
cd ../..
sudo ./build.sh --board example --build-rootfs no
```

### 添加自定义软件包

```bash
# 编辑包列表
vim rootfs/ubuntu/jammy/packages/base.list

# 添加需要的包（每行一个）
vim
htop
tmux
curl
wget

# 重新构建 RootFS
sudo ./build.sh -b example -v ubuntu/jammy -t cli -r yes
```

### 使用本地内核源码

```bash
# 手动放置内核源码
mkdir -p output/linux
cp -r /path/to/your/linux-source/* output/linux/

# 跳过源码下载，直接构建
sudo ./build.sh --board example --kernel linux-6.1 --build-rootfs no
```

### 清理构建产物

```bash
# 清理所有输出
rm -rf output/

# 仅清理 RootFS 缓存
rm output/rootfs-*.tar.gz

# 仅清理内核
rm -rf output/linux output/example-kernel-pkgs/
```

### 并行构建加速

```bash
# 使用所有 CPU 核心
export MAKEFLAGS="-j$(nproc)"
sudo -E ./build.sh --board example --build-rootfs no
```

### 查看构建日志

```bash
# 实时查看构建输出
sudo ./build.sh 2>&1 | tee build.log

# 查看特定阶段的日志
grep "ERROR" build.log
grep "WARNING" build.log
```

## ⚡ 性能优化技巧

### 1. 使用 SSD 存储

将 `output/` 目录放在 SSD 上可显著提升速度。

### 2. 增加交换空间

```bash
# 创建 4GB swap
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

### 3. 使用国内镜像源

国内镜像源可将下载速度提升 5-10 倍。

### 4. 缓存 RootFS

已构建的 RootFS 会自动缓存，二次构建时间从 30 分钟降至 5 分钟。

### 5. 使用 ccache（可选）

```bash
sudo apt-get install ccache
export PATH="/usr/lib/ccache:$PATH"
export CCACHE_DIR="$HOME/.ccache"
```

## ❓ 常见问题

### 构建失败怎么办？

1. 查看错误信息
2. 检查 [常见问题文档](./04-常见问题.md)
3. 清理后重试：`rm -rf output/ && sudo ./build.sh`

### 如何切换镜像源？

重新运行构建脚本，选择不同的镜像源即可。

### 可以同时构建多个配置吗？

可以，不同配置的 RootFS 会独立缓存：
```bash
# Ubuntu Jammy CLI
sudo ./build.sh -v ubuntu/jammy -t cli -r yes

# Ubuntu Jammy XFCE
sudo ./build.sh -v ubuntu/jammy -t xfce -r yes

# Debian Bookworm CLI
sudo ./build.sh -v debian/bookworm -t cli -r yes
```
