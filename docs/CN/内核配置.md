# 内核配置说明

内核配置覆盖文件位于 `configs/target/<target-board>/` 目录。

## 配置查找顺序

构建系统按以下顺序查找内核配置：

1. `configs/target/${TARGET_BOARD}/kernel_defconfig` - 板级 defconfig 覆盖（非空才使用）
2. `arch/${ARCH}/configs/${KERNEL_DEFCONFIG}` - 内核源码配置
3. `output/user_defconfig` - menuconfig 保存的配置
4. `.config` - 内核源码现有配置

## 快速开始

### 方法1：使用内核默认配置

最简单的方式，使用内核源码中的配置。

```bash
# configs/board/myboard.conf
KERNEL_DEFCONFIG="sun55i_t527_bsp_defconfig"

# 构建
./build.sh -b myboard
```

### 方法2：自定义配置（推荐）

```bash
# 1. 进入内核源码
cd output/linux

# 2. 加载基础配置并自定义
make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- sun55i_t527_bsp_defconfig
make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- menuconfig

# 3. 保存为精简配置
make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- savedefconfig

# 4. 复制到 target 目录
cp defconfig ../../configs/target/myboard/kernel_defconfig

# 5. 配置板级文件
# configs/board/myboard.conf
TARGET_BOARD="myboard"
KERNEL_DEFCONFIG="defconfig"

# 6. 构建
cd ../..
./build.sh -b myboard
```

### 方法3：使用 menuconfig 临时调整

```bash
# 构建时启用 menuconfig
./build.sh -b example -k yes

# 这会：
# 1. 加载基础配置
# 2. 启动 menuconfig 让你调整
# 3. 保存到 output/user_defconfig（当 target 和源码 defconfig 都不可用时使用）

# 清除临时配置
rm output/user_defconfig
```

## defconfig 文件

### 文件命名

```
<board>_defconfig           # 标准命名
<board>_<variant>_defconfig # 带变体
```

示例：
- `example_defconfig`
- `t527_minimal_defconfig`
- `t527_full_defconfig`

### defconfig vs .config

| 特性 | defconfig | .config |
|------|-----------|---------|
| 文件大小 | 3-5 KB | 200 KB |
| 内容 | 只有改动项 | 所有配置项 |
| 维护性 | 易维护 | 难维护 |
| 跨版本 | 兼容性好 | 可能不兼容 |
| 推荐度 | 推荐 | 特殊情况使用 |

### 创建 defconfig

```bash
cd output/linux
make ARCH=arm64 menuconfig
make ARCH=arm64 savedefconfig
cp defconfig ../../configs/target/myboard/kernel_defconfig
```

### 使用完整 .config

如果你有完整的 .config 文件：

```bash
cp output/linux/.config configs/target/myboard/kernel_defconfig

# 在板级配置中使用（注意不含 .config 后缀）
TARGET_BOARD="myboard"
KERNEL_DEFCONFIG="defconfig"
```

## 设备树配置

内核设备树覆盖文件位于 `configs/target/<target-board>/kernel.dts`。

### 基本使用

```bash
# 1. 创建设备树文件
# configs/target/myboard/kernel.dts
/dts-v1/;

/ {
    model = "My Board";
    compatible = "vendor,myboard";
    
    memory@40000000 {
        device_type = "memory";
        reg = <0x0 0x40000000 0x0 0x80000000>;
    };
};

# 2. 在板级配置中指定（不含 .dts 后缀）
TARGET_BOARD="myboard"
KERNEL_DTS="myboard"

# 3. 构建
./build.sh -b myboard
```

### 使用子目录

```bash
# 设备树在子目录中
KERNEL_DTS="vendor/allwinner/sun55i-t527-dev"
```

## 配置管理

### 多变体配置

创建多个配置版本：

```bash
configs/target/
├── t527-minimal/kernel_defconfig    # 最小功能
├── t527-standard/kernel_defconfig   # 标准功能
└── t527-debug/kernel_defconfig      # 调试版本

configs/board/
├── t527-minimal.conf   -> TARGET_BOARD="t527-minimal"
├── t527-standard.conf  -> TARGET_BOARD="t527-standard"
└── t527-debug.conf     -> TARGET_BOARD="t527-debug"
```

### 版本控制

```bash
# 提交配置到 Git
git add configs/target/myboard/kernel_defconfig
git commit -m "kernel: update myboard config

- Enable feature X
- Disable feature Y"
```

## 常见问题

### 配置没有生效？

优先检查 `configs/target/${TARGET_BOARD}/kernel_defconfig` 是否存在且非空。
如果 target 文件为空，构建会回退到内核源码中的 `KERNEL_DEFCONFIG`。
`output/user_defconfig` 只在 target 和源码 defconfig 都不可用时作为兜底。

```bash
# 删除临时配置
rm output/user_defconfig
```

### 找不到配置文件？

```bash
# 查看配置
grep KERNEL_DEFCONFIG configs/board/myboard.conf

# 检查文件
ls -l configs/target/myboard/
```

### 如何对比两个配置

```bash
cd output/linux
scripts/diffconfig defconfig1 defconfig2
```
