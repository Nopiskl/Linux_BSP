# 常见问题

## 构建问题

### DNS 解析失败

```bash
sudo mkdir -p /etc/systemd/resolved.conf.d
sudo tee /etc/systemd/resolved.conf.d/dns.conf > /dev/null << 'EOF'
[Resolve]
DNS=8.8.8.8 114.114.114.114
FallbackDNS=223.5.5.5 223.6.6.6
EOF
sudo systemctl restart systemd-resolved
```

### 缺少编译工具链

```bash
sudo apt-get install -y gcc-aarch64-linux-gnu gcc-arm-linux-gnueabihf
```

### 缺少 ccache

ccache 不是必需的，但能加速编译。

```bash
# 安装 ccache
sudo apt-get install -y ccache

# 或禁用 ccache
./build.sh -b example -e no
```

### 权限不足

构建需要 root 权限：

```bash
sudo ./build.sh -b example
```

## 配置问题

### 配置文件找不到

```bash
# 检查配置文件
ls -l configs/board/

# 查看可用板型
./build.sh -b none  # 会列出可用板型
```

### 内核配置没有生效

检查是否存在临时配置：

```bash
# 删除临时配置
rm output/user_defconfig

# 重新构建
./build.sh -b example
```

### 设备树文件找不到

检查文件路径和命名：

```bash
# 查看设备树文件
ls -l configs/target/myboard/

# 在配置中使用（不含 .dts 后缀）
TARGET_BOARD="myboard"
KERNEL_DTS="myboard"
```

## 网络问题

### GitHub 访问慢

使用镜像：

```bash
./build.sh -b example -i https://mirror.ghproxy.com
```

### APT 源速度慢

使用国内镜像：

```bash
# 阿里云
-m https://mirrors.aliyun.com/ubuntu-ports

# 清华
-m https://mirrors.tuna.tsinghua.edu.cn/ubuntu-ports

# 中科大
-m https://mirrors.ustc.edu.cn/ubuntu-ports
```

## 性能问题

### 构建速度慢

1. 使用国内镜像源
2. 启用 ccache（默认已启用）
3. 增加系统交换空间
4. 使用并行编译

```bash
# 查看 CPU 核心数
nproc

# 编译会自动使用 -j$(nproc)
```

### 内存不足

```bash
# 增加交换空间
sudo dd if=/dev/zero of=/swapfile bs=1G count=4
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# 永久挂载
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

### 磁盘空间不足

```bash
# 清理构建产物
rm -rf output/

# 清理 APT 缓存
sudo apt-get clean

# 清理内核源码（如果不需要）
rm -rf output/linux
```

## 源码问题

### 克隆仓库失败

```bash
# 检查网络
ping github.com

# 使用浅克隆
git clone --depth=1 <repo-url>

# 使用 GitHub 镜像
git clone https://mirror.ghproxy.com/https://github.com/<user>/<repo>.git
```

### 源码更新失败

```bash
# 删除旧源码重新获取
rm -rf output/linux
./build.sh -b example -l no
```

## RootFS 问题

### debootstrap 失败

```bash
# 安装 debootstrap
sudo apt-get install -y debootstrap qemu-user-static

# 检查架构支持
dpkg --print-foreign-architectures
```

### 包安装失败

```bash
# 更换镜像源
sudo ./build.sh -b example -r yes -v ubuntu/jammy -t cli \
    -m https://mirrors.aliyun.com/ubuntu-ports

# 检查包名是否正确
apt-cache search <package-name>
```

### 架构不匹配

确保为目标架构选择正确的包和镜像源：

- ARM64: arm64, aarch64
- ARM32: armhf, arm

## 调试技巧

### 查看详细日志

构建脚本会输出详细日志，注意查看错误信息。

### 单独测试组件

```bash
# 仅测试内核构建
./build.sh -b example -o yes

# 仅测试 RootFS 构建
cd output
sudo bash ../tools/build-rootfs.sh -b example -v ubuntu/jammy -t cli
```

### 手动编译内核

```bash
cd output/linux
make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- defconfig
make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- -j$(nproc)
```
