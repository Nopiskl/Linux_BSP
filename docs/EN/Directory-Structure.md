## Main Files

### Top Level Files

- `README.md` - Project documentation, bilingual version
- `build.sh` - Main build script, unified entry

### configs/ Configuration Directory

#### configs/board/

Board configuration files, each file defines complete configuration for a board.

Contains:
- Basic information (board name, architecture)
- Kernel configuration (repository, branch, config file)
- Bootloader configuration
- Device tree configuration
- Boot arguments

#### configs/target/defconfig/

Custom kernel configuration files.

- Higher priority than kernel source configurations
- Minimal configs generated by `savedefconfig`
- Or complete `.config` files

#### configs/target/dts/

Custom device tree files.

- Board-level device trees (.dts)
- Reusable include files (.dtsi)
- Higher priority than kernel source device trees

### tools/ Tools Directory

Collection of build tool scripts.

### rootfs/ RootFS Configuration

Root filesystem configuration and overlay files.

### docs/ Documentation Directory

Complete project documentation in Chinese and English.

### output/ Build Output

Build process output directory (auto-generated).

## Configuration Search Mechanism

### Kernel Configuration Search Order

1. `output/user_defconfig`
2. `configs/target/defconfig/${LINUX_CONFIG}`
3. `configs/target/defconfig/${LINUX_CONFIG}.config`
4. `arch/${ARCH}/configs/${LINUX_CONFIG}`
5. `.config` (kernel source directory)

### Device Tree Search Order

1. `configs/target/dts/${DEVICE_DTS}.dts`
2. `arch/${ARCH}/boot/dts/${DEVICE_DTS}.dts`
